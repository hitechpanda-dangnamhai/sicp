## §1 — T01: CI hard-fail + deploy-drift gate + contract gate

Nhận: W-76 · W-46 · A13(phần gate) · nợ deploy-drift C1.
1. `.github/workflows/ci.yml`: gỡ MỌI soft-fail (`|| echo`, `continue-on-error`)
   — lint/test/typecheck FAIL = pipeline FAIL. Lint phủ workers (nợ KI#5 T02).
2. Coverage gate kiểu RATCHET: đo coverage hiện tại từng app (gateway/web/
   workers/ai/mcp) → set floor = số đo được (làm tròn xuống 1%) → CI fail nếu
   tụt dưới floor. KHÔNG đặt 70% ảo ngay — floor nâng dần mỗi cluster. Ghi
   floor vào file config commit được (`coverage.floors.json`).
3. W-46: gate `pnpm openapi:sync` drift — chạy sync trong CI, `git diff
   --exit-code` trên `packages/shared-types/` → drift = FAIL.
4. Deploy-drift gate (bài học T06 + BACKLOG #32): build-arg `GIT_SHA` stamp
   vào image (gateway/ai/mcp/workers/web) expose qua endpoint health/label;
   script `scripts/smoke-live.sh`: so SHA container vs HEAD + smoke login/
   intent/cart/health → dùng được cả tay lẫn CI-stage deploy (CI chỉ chạy khi
   có môi trường compose — nếu runner không có, stage này điều kiện hoá, ghi rõ).
Acceptance: PR thử có lint-error → CI đỏ · coverage tụt giả lập → đỏ · sửa DTO
không sync → đỏ · `smoke-live.sh` bắt được container cũ (test bằng cách không
rebuild 1 service) · main hiện tại XANH với floor đo thật.
