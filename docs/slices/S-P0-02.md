## Appendix A — ADR drafts cho T01 (mỗi cái ≤30 dòng khi materialize)

**ADR-051 — AI Architecture v6 (umbrella, họ ADR).** Status: Accepted.
Quyết định: kiến trúc AI đích = v5 core (evidence loop · tool-RAG · confidence
deterministic · reflection ×2 · memory 3-tier) + 6 nâng cấp, mỗi nâng cấp sẽ có
ADR con khi C7 chạm: (1) planner tách executor, task-DAG song song — vì latency
và decompose; (2) eval-driven: cấm refactor prompt/graph khi chưa có golden set +
regression gate — vì LLM self-eval không tin được; (3) cost là kiến trúc: routing
theo độ khó + prompt/semantic cache + cost per-tenant; (4) guardrails + per-tool
authz tenant-scoped — biên security mới của multi-tenant AI; (5) tách workload
chat-graph (LangGraph, ADR-048) vs batch/long-running (durable engine, C5/C7);
(6) context budget + compaction per-node. Chi tiết sống ở docs/AI_ARCHITECTURE_v6.md
(Single Home §4); doc này supersede fossil v5. Pattern umbrella theo tiền lệ ADR-037.

**ADR-052 — Re-architecture program (umbrella).** Status: Accepted.
Quyết định: chạy production hoá TẠI sicp (bác greenfield — 0/105 finding có root
cause repo); chương trình 8 cluster C1–C8, thứ tự ép bởi ràng buộc cứng:
W-66 deadline → perimeter P0 → safety-net trước AI-refactor → outbox/DLQ trước
payment → media/cache → tenant depth xuyên suốt → AI v6 → scale infra.
Supersede một phần ADR-045§b (critical path cũ tenant→payment→compliance).
NFR phê chuẩn Human 2026-06-13, giá trị sống tại BACKLOG §1. Sổ phủ:
INVENTORY_2026-06-13 map 105→cluster tại BACKLOG §3, luật W-ID mồ côi trong CLAUDE.md.

**ADR-053 — Media: ảnh ra object store + CDN.** Status: Accepted.
Bối cảnh: V010 inline base64 trong Postgres (không có ADR) = one-way door —
phình DB, chặn CDN, thổi payload, vô hiệu next/image (W-90, W-47). Quyết định:
ảnh → object store (S3-compatible; MinIO dev / managed prod) + serve qua CDN;
DB chỉ giữ key+metadata; migration backfill 2 pha (dual-read → cutover).
Thi hành: C4. Trade-off: thêm 1 hạ tầng, đổi lấy mở khoá toàn bộ tầng perf media.

**ADR-054 — LLM cost architecture.** Status: Accepted.
Quyết định: (1) giữ provider-abstraction 1 điểm swap (đã có — INVENTORY ⭐7);
(2) W-93 durable trace input/output/score per-call ngay từ C2 — vừa là eval data
vừa là dataset distill; (3) route task classification/extraction sang model lite
(W-92); (4) lộ trình distill/fine-tune model nhỏ self-host (vLLM) cho task rẻ
SAU KHI eval harness sống (phụ thuộc cứng: không thước đo = không train);
(5) cost metric per-tenant per-intent (W-40/W-55). GPU self-host chỉ duyệt khi
data chứng minh break-even vs API — quyết định bằng số, không bằng niềm tin.

**ADR-055 — Stock concurrency: optimistic atomic decrement.** Status: Accepted.
Quyết định: trừ kho = single-statement atomic `UPDATE ... WHERE stock_qty >= n`
(tenant-scoped), KHÔNG `SELECT FOR UPDATE` (giữ lock ngắn nhất, không ôm lock
qua external call), KHÔNG trừ kho ở validate (advisory only). Invariant
xuyên cart/checkout/payment: stock_qty không bao giờ âm, enforced tại DB
statement, không tại app logic. Flash-sale pattern (Redis pre-deduct) = P2
khi có số liệu tải; ghi chú để không ai "nâng cấp" sớm.
