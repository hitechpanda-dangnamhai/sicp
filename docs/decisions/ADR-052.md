# ADR-052 — Re-architecture program (umbrella)

- **Status:** Accepted
- **Date:** 2026-06-13
- **Refs:** ADR-045 (partial supersede §b critical-path cũ), ADR-037 (pivot production). Nguồn: `docs/INVENTORY_2026-06-13.md` (105 mục RECON audit).
- **Context:** RECON audit 3 phiên (S-SCALE-AUDIT 22 verdict + S-WEAKNESS W-23→84 + S-BLINDSPOT W-85→105) = **105 mục**. 0/105 finding có root cause "repo sai từ gốc" → KHÔNG greenfield.
- **Decision:**
  - **Production hoá TẠI `sicp`** (bác greenfield — 0/105 finding biện hộ rebuild from scratch).
  - **Chương trình 8 cluster C1–C8**, thứ tự ÉP bởi ràng buộc cứng:
    **W-66 deadline (2026-08-01) → perimeter P0 → safety-net TRƯỚC AI-refactor → outbox/DLQ TRƯỚC payment → media/cache → tenant depth xuyên suốt → AI v6 → scale infra.**
  - Map 105→cluster sống tại `MASTER_BACKLOG.md §3` (sổ phủ, luật W-ID mồ côi trong CLAUDE.md §5: đóng slice phải tick W-IDs nó nhận).
  - **NFR phê chuẩn Human 2026-06-13**, giá trị sống tại `BACKLOG §1` (KHÔNG copy vào ADR — Single Home).
- **Rationale:** Thứ tự không phải ưu tiên chủ quan mà là dependency cứng: W-66 có deadline lịch; perimeter hở thì mọi tầng trên vô nghĩa; refactor AI không safety-net = mù; payment cần outbox/DLQ trước nếu không charge-fail mất dấu; tenant depth phải xuyên suốt mọi cluster.
- **Trade-offs:** Supersede **một phần ADR-045§b** (critical path cũ tenant→payment→compliance) — tenant vẫn xuyên suốt nhưng perimeter P0 + safety-net chèn TRƯỚC. ADR-045§a (Workflow v2 dual-surface) giữ nguyên.
- **Reference:** `docs/INVENTORY_2026-06-13.md` (evidence read-only); `MASTER_BACKLOG.md §1` (NFR) + `§3` (map 105); cluster C1 = slice S-P0-02.
