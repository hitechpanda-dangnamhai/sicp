# ADR-047 — AI service trusts Gateway-forwarded identity (no JWT verify in AI/MCP)
- **Status:** Accepted
- **Date:** 2026-06-11
- **Refs:** ADR-003 (MCP single I/O), ADR-019 (cookie auth), ADR-040 + ADR-046 (tenant resolve)
- **Context:** Code thật (Gateway `JwtAuthGuard`, AI service `apps/ai/src/main.py` đọc `X-User-Id` Gateway-forward, MCP tool `auth.verify_jwt` STUB trả None) đã chốt pattern security boundary nhưng chưa LOCK ADR. Trước khi build slice multi-tenant (V011) cần ghi rõ vì tenant resolve cũng đi qua boundary này (Gateway resolve → AI/MCP tin), và đường go-live phụ thuộc hợp lệ của perimeter.
- **Decision:**
  - **Gateway = security perimeter duy nhất:** verify JWT cookie qua `JwtAuthGuard`; resolve `user_id` (V001) + `tenant_id` (V011); forward qua HTTP header `X-User-Id` (+ `X-Tenant-Id` sau V011) tới AI service. Trace context propagate cùng.
  - **AI service + MCP server KHÔNG verify JWT** — tin header Gateway forward. State (`IcpState`) chứa `user_id` (và `tenant_id` sau V011) đã được resolve.
  - **MCP tool `auth.verify_jwt` = STUB legacy** (trả None) — KHÔNG dùng trong AI graph. Giữ để không phá registry, đánh dấu deprecated khi tiện; KHÔNG xoá vào commit khác.
  - **Hệ quả vận hành:** AI service (`apps/ai`) + MCP server (`apps/mcp`) CẤM expose public; chỉ chấp nhận traffic từ Gateway. Enforce ở infra (Docker network internal-only; production = Kubernetes NetworkPolicy / VPC private subnet — KHÔNG ingress).
- **Rationale:** (a) JWT verify 2 lần (Gateway + AI) là redundant + tăng latency + risk drift secret giữa service. (b) Single perimeter dễ audit hơn (1 nơi kiểm credential). (c) Pattern code đã chốt — ADR này chỉ chính thức hoá để slice tenant + audit + hardening dùng đúng boundary.
- **Trade-offs:** Bypass Gateway = bypass auth toàn bộ → Network policy isolation BẮT BUỘC (KHÔNG có defense-in-depth ở app layer). Tenant slice (V011) phải thêm `X-Tenant-Id` vào forward header pattern + Gateway `TenantContextGuard` resolve trước khi forward (ADR-046).
- **Reference:** `apps/gateway/src/auth/jwt-auth.guard.ts` + `apps/ai/src/main.py` (forward header read); `docs/00_CONTEXT.md` §2 service boundary table; ADR-003 (MCP single I/O) + ADR-019 (cookie auth) + ADR-040 + ADR-046 (tenant resolve at Gateway); infra: `infra/docker-compose.yml` network `icp` internal.

## Amendment 2026-06-13 #2 (S-P0-02/T04) — ingress isolation = gỡ published ports (KHÔNG internal:true)
Thi hành thực tế phát hiện `internal: true` KHÔNG khả thi: AI (Gemini/OpenAI) +
MCP (Gemini vision/gtrends) CẦN **egress internet**; `internal:true` chặn luôn
egress → vỡ LLM/vision → vỡ SSE end-to-end. Threat-model thật của ADR-047 =
**chặn INGRESS** (host/spoof không tới được AI/MCP). Đạt đúng mục tiêu đó bằng:
**GỠ published ports** (`infra/docker-compose.yml`: bỏ `5001:5001`/`5050:5050`)
→ AI/MCP chỉ reachable từ service cùng network nội bộ (Gateway), KHÔNG từ host.
`docker-compose.dev.yml` override tái expose cho debug local (KHÔNG dùng prod).
**Residual risk** (container khác cùng network vẫn tới AI/MCP) + **egress-control
allowlist + mTLS service-to-service = C8**. Verified live: `curl host:5001/:5050`
→ connection refused; gateway→AI nội bộ + SSE vẫn chạy.

## Amendment 2026-06-13 (S-P0-02/T01) — isolation phải ENFORCE thật, không chỉ trên giấy
"Network isolation infra-level" chuyển từ **khuyến nghị sang BẮT BUỘC enforce**:
compose `internal: true` cho network AI/MCP + **KHÔNG expose port AI/MCP ra host**
(hiện `infra/docker-compose.yml`: ai `5001:5001`@214, mcp `5050:5050`@243, network
`icp` đơn KHÔNG internal → port lộ host = ADR-047 chỉ trên giấy). Thi hành:
S-P0-02/T04 (compose internal + gỡ ports + override dev giữ port debug). mTLS
service-to-service = **C8**. **Trust header CHỈ hợp lệ khi isolation đang enforce —
không isolation = không trust** (nếu không, bypass Gateway = bypass toàn bộ auth).
