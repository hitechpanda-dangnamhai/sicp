# ADR-056 — CI posture cho LLM-driven e2e: non-blocking live job

Status: Accepted
Liên quan: ADR-052 (program C2 safety-net) · ADR-054 (cost) · ADR-051 (eval-driven) · S-P0-03 §0.5

## Context
- e2e LLM-driven (intent qua Gemini/OpenAI) non-deterministic + <8s/intent + phụ thuộc quota/mạng.
- §0.5 slice (S-P0-03.md): gate CI làm ĐỎ main do flake = đỏ giả, cấm.
- Muốn coverage full-stack thật (browser→gateway→AI→MCP→SSE→UI) với LLM THẬT, không chỉ mock.
- T04 eval đã có mock/live split ở graph-level (bổ sung, không thay e2e wiring).

## Decision
1. e2e DETERMINISTIC (T02b-1: auth/cart/isolation, 0 LLM) = HARD gate (fail=pipeline fail).
2. e2e LLM-DRIVEN (T02b-2: 6 intent) = job CI NON-BLOCKING: chạy live, assert STRUCTURAL (reach
   result-state/SSE-complete, KHÔNG assert content LLM), report (annotation/artifact), KHÔNG fail
   pipeline khi flake. Keys qua CI secret.
3. Regression intent/tool-call BẮT bằng T04 eval (mock hard-gate + live-manual) — e2e live chỉ
   chứng wiring sống, không là cổng chặn merge.

## Consequences
- (+) CI xanh-tin (không đỏ giả do LLM flake) — tôn §0.5.
- (+) Tín hiệu live e2e mỗi run cho mắt người soi.
- (−) LLM-flake không chặn merge → dựa T04 eval làm gate chất lượng intent. Report toothless →
  revisit: nightly-required hoặc threshold-based.
- (−) Keys LLM trong CI secret = cost mỗi run + bề mặt bảo mật (cần quota cap + budget alert).
