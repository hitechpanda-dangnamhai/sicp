# ADR-040 — Multi-tenant SaaS (tenant isolation)
- **Status:** Accepted
- **Date:** 2026-06-09
- **Supersedes:** phần "multi-tenant" bị cut trong ADR-010
- **Context:** Sản phẩm production phục vụ nhiều shop trên cùng hệ thống.
- **Decision:** Hệ thống là **SaaS multi-tenant**. Mọi dữ liệu thuộc về một `tenant_id`; isolation bắt buộc ở tầng DB (cột `tenant_id` + index, ưu tiên Row-Level Security), cache (key có tenant), search (filter tenant trong Vespa), và events (tenant trong payload + Kafka header). JWT mang `tenant_id`; trace context propagate `tenant_id`. **Không bao giờ** truy vấn cross-tenant.
- **Rationale:** Isolation là yêu cầu nền tảng của SaaS; nhúng từ schema sớm rẻ hơn retrofit về sau.
- **Trade-offs:** Migration lớn (thêm `tenant_id` gần như mọi bảng), mọi query phải scoped, tăng độ phức tạp test. Đây là thay đổi nặng nhất của pivot.
- **Reference:** `docs/00_CONTEXT.md` §3 pillar 10 (multi-tenant + GDPR) + `docs/MASTER_BACKLOG.md` §3 P0 #1 (RLS + tenant_id) + V011 spec: `docs/archive-v1/02_DATA_MODEL.md` §1.X (lớn nhất); ADR-046 (model #2) + ADR-047 (Gateway perimeter); legacy spec: `docs/archive-v1/01_ARCHITECTURE.md`, `docs/archive-v1/03_API_CONTRACTS.md`, `docs/archive-v1/06_OBSERVABILITY.md`, `docs/archive-v1/07_BEHAVIOR_LOGS.md`, `docs/archive-v1/09_FIELD_AUDIT.md`.

## Amendment 2026-06-11 — Enforcement chi tiết (S-P0-01, RECON verified)
Bối cảnh: role `icp` hiện là superuser+BYPASSRLS → RLS vô hiệu; Gateway raw pg
pool / MCP psycopg connection-per-query (không pooler chen giữa) → SET LOCAL khả thi.
Chốt 4 chi tiết enforcement (LOCKED, cross-slice):
(i)   Role runtime `icp_app` NOBYPASSRLS; mọi query data path chạy trong txn có
      `SET LOCAL app.tenant_id = $tenant`; RLS policy `tenant_id =
      current_setting('app.tenant_id')::uuid` trên mọi bảng tenant-scoped.
(ii)  Tách 2 connection string: `DATABASE_URL_MIGRATE` (superuser, chỉ
      apply.sh) vs `DATABASE_URL` (icp_app, runtime). Service boot bằng
      superuser = lỗi cấu hình.
(iii) Vespa: mọi YQL build qua helper trung tâm (apps/mcp) tự inject
      `tenant_id` filter; CẤM tool gọi vespa query raw. Test bắt buộc:
      cross-tenant search trả 0 row.
(iv)  Redis key prefix tenant: `idem:cache:{tenant}:{user}:...`,
      `sse:pubsub:{tenant}:{rid}`, RedisSaver thread_id =
      `{tenant}:{rid}`. Key thiếu tenant trong data path = bug.
