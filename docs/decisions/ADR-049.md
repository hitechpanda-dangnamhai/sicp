# ADR-049 — Idempotency middleware apply per-route (not global) — login password attempts MUST NOT cache
- **Status:** Accepted
- **Date:** 2026-06-11
- **Refs:** ADR-004 (Redis SETNX lock + cache pattern), ADR-048 (intent-action attempt_n)
- **Context:** ADR-004 LOCK Redis SETNX lock (30s) + response cache (24h) pattern nhưng KHÔNG nói rõ SCOPE apply. Code thật đã chốt apply **per-route**, KHÔNG global. Khi build route mutating mới (vd tenant onboarding V011, payment IPN, refund), agent phải explicit `.forRoutes()` đăng ký — nếu quên = mất idempotency = side-effect lặp được (IPN duplicate, double-charge).
- **Decision:**
  - **Base `IdempotencyMiddleware`** (24h cache) apply per-module qua `consumer.apply(...).forRoutes(...)`. Route hiện đã đăng ký: `POST /intent` · `POST /products` · `PATCH /products/:id` · `POST /orders/checkout` · `POST /cards/:id/{accept,reject}` · 6 cart write route (`POST/PATCH/DELETE /cart/items*` + `DELETE /cart` + `POST/DELETE /cart/promo`).
  - **`IntentActionIdempotencyMiddleware`** riêng cho `POST /intent/:rid/action` — TTL **5min** (≠ 24h base), key composite `intent:action:{rid}:{attempt_n}` cho retry semantics (xem ADR-048).
  - **CẤM cache `/auth/login` + password reset attempts** — đạt được bằng cách **KHÔNG liệt kê** trong `.forRoutes()` (KHÔNG dùng `.exclude()`). Lý do: cache password attempts = replay attack window (attacker reuse Idempotency-Key trong 24h → re-validate same password). `/auth/refresh` cũng KHÔNG cache (mỗi refresh phải rotate fresh).
  - **Default = OFF:** route mutating mới phải explicit đăng ký. KHÔNG có blanket "mọi POST/PUT/PATCH/DELETE auto-idempotent".
- **Rationale:** Per-route scope cho phép tinh chỉnh TTL theo loại operation (24h cho order/intent commit, 5min cho intent action retry, OFF cho auth). Global apply sẽ ép pattern không phù hợp lên auth/health/track.
- **Trade-offs:** Risk = quên đăng ký route mới = mất idempotency (side-effect lặp). Mitigation: **code review checklist** mới — mọi PR thêm route POST/PUT/PATCH/DELETE phải verify `.forRoutes()` đã wire (hoặc cố ý exempt như auth). Đề xuất add lint rule khi tiện (chưa code).
- **Reference:** `apps/gateway/src/idempotency/idempotency.middleware.ts` (base, 24h); `apps/gateway/src/intent/intent-action-idempotency.middleware.ts` (5min, attempt_n); per-module `.forRoutes()` calls trong `intent.module.ts` / `cart.module.ts` / `cards.module.ts` / `products.module.ts`. ADR-004 (lock + cache lifecycle); ADR-048 (attempt_n composite key).
