# ADR-050: Per-intent membership policy tại Gateway (default-deny)

Status: Accepted · 2026-06-12 · Refs: ADR-046, ADR-047, T03d item 5

CONTEXT: Guard đồng nhất (JwtAuthGuard+TenantMembershipGuard) tại
POST /intent + POST /:rid/action chặn customer 0-membership (BY DESIGN,
ADR-046) khỏi MỌI intent — storefront không dùng được cho khách.
Dispatch map (modality, entry_intent)→graph sống ở AI (main.py:326-398)
SAU perimeter; AI không có membership info (ADR-047); router thuần
modality, KHÔNG LLM (router_graph.py:68-72); tuple body client-supplied.

DECISION:
1. Enforcement tại Gateway (perimeter): IntentPolicyGuard thay
   TenantMembershipGuard ở cả 2 endpoint.
2. Matrix v1: membership-required = {01 import, 07 analyzing};
   customer-allowed (tenant strict + ownership T03c) = {02 buy,
   03 search, 04 recommend, 05 cart}.
3. DEFAULT-DENY: tuple không khớp danh sách customer-allowed → đòi
   membership. WHY: hai bản đồ hai ngôn ngữ (TS guard / Python
   dispatch) tất drift — chiều fail này biến drift thành bug nhìn
   thấy (customer 403 oan, có ticket) thay vì lỗ security im lặng.
4. Policy tính 1 lần lúc POST, ghi membership_required vào VALUE
   intent:cache (cùng entry ownership T03c, thừa kế TTL 60s);
   /:rid/action đọc từ cache sau ownership-check, KHÔNG parse lại body.
5. Matrix sống trong CODE + table-driven test — KHÔNG dùng bảng
   policies (DSL đó scope event→action, glossary 00_CONTEXT §5;
   authz trong DB mutable = thêm bề mặt tấn công).

NGOÀI MATRIX: intent 06 payment = REST riêng, authz thuộc scope S-06;
intent 08 auth = Gateway auth controller. Matrix này KHÔNG phủ đường
payment.
