# ADR-048 — LangGraph interrupt+resume + RedisSaver checkpoint + Redis pub/sub SSE forward (Pattern A)
- **Status:** Accepted
- **Date:** 2026-06-11
- **Refs:** ADR-007 (single Flask + LangGraph), ADR-019 (cookie httpOnly SSE auth)
- **Context:** Code thật (`searching_by_text.py`, `cart_by_text.py`, `importing_by_images.py`, `buying_by_voices.py`, `recommend_by_images.py`, `analyzing_by_voices.py` — 6 graph) đã chốt pattern interrupt+resume + SSE forwarding nhưng chưa LOCK ADR. Pattern tái dùng cross 6 slice (S-04 search, S-05 cart, S-07 import, S-08 voice buy, S-09 recommend, S-10 analytics) và sẽ chặn S-P0-01 (multi-tenant V011 phải forward `X-Tenant-Id` cùng pattern).
- **Decision:** Pattern A gói 3 element + 2 ngoại vi:
  - **(a) LangGraph interrupt+resume:** node gọi `interrupt({awaiting: <token>})` → graph PAUSE → FE `POST /api/v1/intent/{rid}/action {choice, value, _meta:{attempt_n}}` → AI service `graph.astream(Command(resume=<choice>), config={'configurable':{'thread_id': rid}})` → graph resume từ checkpoint. Pattern P2 dynamic (interrupt only when condition matches: typo confidence >0.85 / LLM TimeoutError / rank_finalize cart action).
  - **(b) RedisSaver checkpointer:** key `intent:checkpoint:{rid}` TTL **30min** + `refresh_on_read=True` (Strategy β). Explicit `await saver.adelete_thread(rid)` sau `final` event = fast-path cleanup. Package `langgraph-checkpoint-redis`.
  - **(c) Option Z Redis pub/sub SSE forward:** AI service `redis.publish('sse:pubsub:{rid}', json)`; Gateway `intent.controller.ts` open EventSource → `ioredis.duplicate().subscribe('sse:pubsub:{rid}')` → forward mỗi message qua `res.write(...)` (manual SSE, KHÔNG Nest `Observable`/`@Sse`). Pub/sub ephemeral (no TTL, no buffer). Cleanup: `final` event → Gateway close + unsubscribe; FE close (`req.on('close')`) → unsubscribe; 60s idle → unsubscribe + implicit `Command(resume={choice:'skip'})` tới AI internal `/resume`.
  - **(d) attempt_n composite idempotency:** `_meta.attempt_n` monotonic int FE → Gateway `IntentActionIdempotencyMiddleware` cache key `intent:action:{rid}:{attempt_n}` TTL **5min** (≠ 24h base). Cùng key + khác attempt_n = dispatch mới (retry hợp lệ).
  - **(e) choice enum** semantic per use case: typo `{accept|reject}`, degrade `{retry_ai|continue_basic}`, cart_action `{add_to_cart|skip}`, clarify `{clarify_pick}`. Resume tới node gốc đã pause.
- **Rationale:** (a) interrupt+resume = thay thế multi-endpoint clarify flow (1 SSE connection cho cả request). (b) RedisSaver = checkpoint nhỏ + TTL refresh = không kẹt graph khi user idle vài phút. (c) Option Z = AI publish độc lập Gateway, scale horizontally; Gateway chỉ forward (stateless). (d) attempt_n = phân biệt retry hợp lệ vs replay attack.
- **Trade-offs:** Pub/sub ephemeral → FE reconnect = mất event đã publish trong gap (chấp nhận, không buffer). RedisSaver TTL 30min → user idle >30min mất state (chấp nhận, tradeoff vs memory cost). Pattern A yêu cầu mọi graph có node `interrupt()` phải emit SSE `status:awaiting_user_input` TRƯỚC interrupt (FE biết cần action).
- **Reference:** `apps/ai/src/main.py` graph.astream + Command(resume); `apps/ai/src/tools/redis_publisher.py`; `apps/gateway/src/intent/intent.controller.ts:259-365`; `apps/gateway/src/intent/intent-action-idempotency.middleware.ts`; 6 graph files. Reuse evidence: S-04/S-05/S-07/S-08/S-09/S-10.
