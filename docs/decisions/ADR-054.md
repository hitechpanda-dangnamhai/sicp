# ADR-054 — LLM cost architecture

- **Status:** Accepted
- **Date:** 2026-06-13
- **Refs:** ADR-051 (AI v6 umbrella, nâng cấp #3 cost), ADR-052 (program). Findings: W-40, W-55, W-92, W-93, W-33 (INVENTORY ⭐7).
- **Context:** Provider-independence ĐÃ xuất sắc (1 điểm swap LLM + CLIP self-host $0 — INVENTORY insight #7). Thiếu = durable trace cost + route task rẻ sang model lite. Đòn bẩy giảm-cost gần sẵn sàng.
- **Decision:**
  1. **Giữ provider-abstraction 1 điểm swap** (đã có — INVENTORY ⭐7, KHÔNG đổi).
  2. **W-93 durable trace** input/output/score per-call **ngay từ C2** — vừa eval data vừa dataset distill.
  3. **Route task** classification/extraction sang **model lite** (W-92).
  4. **Lộ trình distill/fine-tune** model nhỏ self-host (vLLM) cho task rẻ — CHỈ SAU KHI eval harness sống (phụ thuộc cứng: không thước đo = không train).
  5. **Cost metric per-tenant per-intent** (W-40/W-55).
- **Rationale:** Cost là kiến trúc, không phải tối ưu sau. Durable trace (#2) là tiền-điều-kiện cho cả eval lẫn distill — làm sớm (C2) để có data tích luỹ. GPU self-host CHỈ duyệt khi data chứng minh break-even vs API.
- **Trade-offs:** Quyết định bằng số (break-even data), KHÔNG bằng niềm tin. Distill defer tới sau eval harness — chấp nhận trả API cost trong lúc tích data.
- **Reference:** `llm_client.py` (provider abstraction + usage_metadata chưa đọc — W-40); thi hành trace từ **C2**, route lite + cost-metric tại **C7** (AI v6).
