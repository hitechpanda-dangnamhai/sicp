# ADR-053 — Media: ảnh ra object store + CDN

- **Status:** Accepted
- **Date:** 2026-06-13
- **Refs:** ADR-052 (program, cluster C4/C5 media), ADR-036 (CLIP embeddings Vespa-only). Findings: W-90, W-47 (INVENTORY).
- **Context:** V010 inline base64 ảnh trong Postgres (`products.image_data`, KHÔNG có ADR quyết định) = **one-way door**: phình DB, chặn CDN, thổi payload, vô hiệu `next/image` (W-47, W-90). Postgres trở thành image server.
- **Decision:**
  - Ảnh → **object store** (S3-compatible: MinIO dev / managed prod) + serve qua **CDN**.
  - DB chỉ giữ **key + metadata** (KHÔNG giữ bytes).
  - Migration backfill **2 pha**: dual-read (đọc cả base64 cũ + object store mới) → cutover (xoá cột base64 sau khi backfill xong + verify).
- **Rationale:** Đòn bẩy cao nhất cho tải hạ nguồn (insight #6 INVENTORY): càng để lâu base64 càng đắt gỡ. Tách bytes khỏi DB mở khoá toàn bộ tầng perf media (CDN, image-opt FE, payload nhỏ).
- **Trade-offs:** Thêm 1 hạ tầng (object store) — đổi lấy mở khoá tầng perf. CLIP embeddings vẫn Vespa-only (ADR-036, KHÔNG đổi).
- **Reference:** Thi hành **C4** (media/cache cluster). `products.py:26`, `V010 image_data` (nguồn hiện trạng); backfill = 2-phase dual-read→cutover.
