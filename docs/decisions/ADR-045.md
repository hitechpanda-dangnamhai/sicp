# ADR-045 — Workflow v2 (dual-surface) + critical-path order tenant-before-payment
- **Status:** Accepted
- **Date:** 2026-06-10
- **Context:** Vòng đời docs cũ stale tái phát: fact sống nhiều nhà, status copy tay, mâu thuẫn ADR↔code. Đồng thời quyết định P0 order cho production cần ghi nhận chính thức trước khi cắt slice tenant/payment.
- **Decision:**
  - **(a) Vận hành theo Workflow v2:** dual-surface (Claude Web plan / Claude Code execute); `docs/FACTS.md` máy sinh làm cầu nối + CI gate facts-drift; nguyên tắc PULL (tri thức triển khai truy xuất từ code, không chép sẵn); Single Home (status chỉ sống ở `MASTER_BACKLOG.md`, không ở ADR); luật commit squash-theo-task (1 task = 1 commit chuẩn `^(S-\d+\/(T\d+|HOTFIX-\d+|REFACTOR-\d+)|META): .+`). Ref: `docs/ICP_WORKFLOW_V2_WORKING.md` + `docs/ICP_PLAYBOOK_V2.2.md`.
  - **(b) Critical path P0:** **MULTI-TENANT (V011) TRƯỚC PAYMENT**. Toàn bộ V011 còn lại (payment ALTER, GDPR, audit, LTR) chạy sau khi tenant_id đã nhúng schema.
- **Rationale:**
  - (a) Diệt gốc rễ docs stale: thay vì gác bằng kỷ luật con người (luôn thất bại), gác bằng máy (gen-facts + CI). PULL+Single Home đảm bảo hai surface không drift.
  - (b) Tenant chạm schema gần như mọi bảng. Build payment trước rồi retrofit `tenant_id` vào dữ liệu tài chính = rủi ro pháp lý + chi phí migration backfill lớn nhất có thể. Hiện tại là điểm rẻ nhất để chuyển vì 0 user thật, `tenant_id` = 0 toàn DB.
- **Trade-offs:** Workflow v2 yêu cầu kỷ luật mở task = đọc CLAUDE.md + cập nhật FACTS trước khi đóng slice (tốn ~5 phút/slice — đã chấp nhận). Tenant trước payment kéo dài timeline tới go-live thêm 1 slice nhưng tiết kiệm 1 hotfix tenant cho payment sau này.
- **Reference:** `CLAUDE.md` (luật rút gọn) · `docs/ICP_WORKFLOW_V2_WORKING.md` · `docs/ICP_PLAYBOOK_V2.2.md` · `scripts/gen-facts.sh` · `.github/workflows/guards.yml` · ADR-040 (multi-tenant) · ADR-038 (payment).
