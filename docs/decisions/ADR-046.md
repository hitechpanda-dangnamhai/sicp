# ADR-046 — Tenant data model #2 — marketplace (users/sessions global)
- **Status:** Accepted
- **Date:** 2026-06-11
- **Refs:** ADR-040 (làm rõ, không supersede)
- **Context:** ADR-040 LOCK "SaaS multi-tenant" và "mọi dữ liệu thuộc về một `tenant_id`" nhưng KHÔNG nói rõ `users`/`sessions` có `tenant_id` hay không. Trước khi build slice multi-tenant (V011) cần chốt model dữ liệu để tránh phải retrofit `users.tenant_id` sau khi đã có khách thật.
- **Decision:** Chọn **model #2 — marketplace**. Cụ thể:
  - `users` + `sessions` = **account GLOBAL**, KHÔNG có cột `tenant_id` (1 customer có thể mua nhiều shop bằng cùng 1 account).
  - Quan hệ user ↔ tenant cho **merchant/staff** ghi qua bảng riêng `tenant_memberships(user_id, tenant_id, role)` (RBAC quản trị shop).
  - **Customer global KHÔNG có membership** — quyền mua hàng là quyền global.
  - **Mọi bảng dữ liệu khác** (products, orders, transactions, events, behavior_events, action_cards, policies, insights, ...) tenant-scoped: thêm cột `tenant_id NOT NULL`, enforce isolation 2 lớp — Row-Level Security ở DB + scope app-level ở repo/port.
  - **`tenant_id` resolve per-request** ở Gateway (X-Tenant-Id header hoặc subdomain/path), KHÔNG suy từ user.
- **Rationale:** (a) Khớp UX commerce phổ thông VN (Shopee/Lazada: 1 tài khoản, nhiều shop). (b) Tránh duplicate `users` cross-shop. (c) Model #1 (users gắn 1 tenant) sẽ buộc fork account khi customer mua shop thứ 2 — không phù hợp.
- **Trade-offs:** Hai tầng quyền (global = mua hàng, scoped = quản trị) → cần kiểm 2 nhánh. RLS policy phải bypass cho bảng `users`/`sessions`; logic membership lookup thêm 1 query khi merchant/staff đăng nhập.
- **Reference:** `docs/00_CONTEXT.md` §3 pillar 10 + §5 Glossary; ADR-040 (umbrella multi-tenant); V011 (target migration) — kèm bảng `tenants`, `tenant_memberships`, thêm `tenant_id` 10 bảng tenant-scoped, RLS policies.

## Amendment 2026-06-11 (b) — Deploy model + anonymous tenant resolution

**Deploy model V1:** path prefix `icp.vn/s/<slug>/...`. Custom domain (Cách 3)
defer P1 sau go-live. Subdomain (Cách 1) skip.

**Anonymous tenant resolution chain (Gateway, mọi public endpoint nhận
behavior/track/public-search):**
1. JWT cookie có `tenant_id` → dùng (authed).
2. Header `X-Tenant-Id` (FE attach, FE đã resolve qua `/api/v1/public/tenant-by-slug/<slug>`).
3. Cả 2 thiếu → `400 TenantContextMissing`. CẤM silent drop. CẤM persist tenant_id=NULL.

**FE responsibility:** resolve slug từ pathname → cache tenant_id session →
attach header mọi anonymous request. Mobile dùng cùng pattern (deep link/onboarding).

**BE responsibility:** KHÔNG parse Host/Referer để resolve tenant — chỉ tin
header (cho anonymous) hoặc JWT (cho authed).

**Sentinel "system" tenant:** CHỈ cho job scheduler/admin cross-tenant ops.
KHÔNG dùng cho FE event.

## Amendment 2026-06-11 (c) — Active tenant = URL, JWT = membership list

**Supersedes phần "active-tenant JWT" của amendment 1.**

**Active tenant pattern (LOCKED):** URL `/s/<slug>` là source of truth duy nhất cho tenant context của mọi request từ FE. Không có "active tenant claim" trong JWT, không có "active tenant state" trong session phục vụ request routing.

**JWT shape:**
- `tenant_ids: string[]` — danh sách tenant user là member.
- Refresh re-lookup memberships, không cache active.

**Session DB (`sessions.last_active_tenant_id`):** CHỈ dùng cho landing khi user vào root URL không có `/s/<slug>`. Update khi switch-tenant. KHÔNG dùng cho request routing.

**POST /auth/switch-tenant:**
- Verify membership.
- UPDATE sessions.last_active_tenant_id.
- Return `{ tenant_id, slug, redirect_url }`. KHÔNG re-issue token.

**Guard chain mỗi request:**
1. TenantResolverService (amend b) → tenant_id từ URL/header.
2. TenantMembershipGuard: check tenant_id IN jwt.tenant_ids → 403 nếu không.

**Lý do chọn pattern này (Linear/Vercel/GitHub):** multi-tab tự nhiên, deep-link sống được, single source of truth URL, không sync state, refresh không quan tâm active.

**Bổ sung 2026-06-11:** `GET /api/v1/auth/tenants` trả `tenant_ids[]` của user — FE cần endpoint này cho onboarding/switcher vì httpOnly JWT không decode được client-side.
